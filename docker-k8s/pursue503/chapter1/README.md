# 1-2 도커를 사용하는 의의

- 변화하지 않는 실행 환경으로 멱등성 확보
- 코드를 통한 실행 환경 구축 및 애플리케이션 구성
- 실행 환경과 애플리케이션의 일체화로 이식성 향상
- 시스템을 구성하는 애플리케이션 및 미들웨어의 관리 용이성

1. A 서버 , B서버에 배포했을시 다르게 동작하는 문제
2. 서버에 설치된 라이브러리 버전이 달라서 생김
3. 코드로 관리하여 불변인프라를 생성할 수 있음(도커를 사용하면)
